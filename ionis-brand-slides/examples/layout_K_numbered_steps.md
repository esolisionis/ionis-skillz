# Example — Layout K (Numbered step row)

A worked example of the 3-step process layout, with the canonical Raspberry → Magenta → Orange color sequence.

---

### Slide 6 — The Trigger

**Layout:** Layout K (Numbered step row, 3 steps)

**Title:** "How the AI decides when to call a skill."

**Body / copy regions:**

- **Lede above the steps:** "It's a matching problem, not a magic problem. The model sees the user's request, then scans skillz *descriptions* for a fit."

- **Step 1 (Raspberry top bar, Raspberry numbered circle "1"):**
  - Headline: "You ask."
  - Body: "'Build me a deck for the Q3 review.' The model now has a task."

- **Step 2 (Magenta top bar, Magenta numbered circle "2"):**
  - Headline: "It scans."
  - Body: "The model reads each skill's `description` — never the full body. Cheap, fast."

- **Step 3 (Orange top bar, Orange numbered circle "3" with dark text):**
  - Headline: "It loads."
  - Body: "If a description matches the task, the skill's instructions and resources are pulled into context for that run."

- **Quote-pull band below the steps (solid Pink tint background, Orange left border):**
  - "The description is the **contract**. A good one says exactly when to use the skill — and just as importantly, when *not* to."

**Visual spec:**
- Primary color anchor: Raspberry (anchors the sequence as Step 1, plus the slide title's solid Raspberry rule)
- Prow block composition: none (Layout K uses rectangular cards with top color bars + numbered circles, not Prows)
- Wing treatment: Cool Gray 1 watermark, left edge, alt-cropped, ~14% opacity
- Photography: none
- Typography accents: solid Raspberry rule under the slide title; quote-pull `<strong>` text in Raspberry

**Color sequence note:** The Raspberry → Magenta → Orange progression is walked across the steps as **three separate solid colors**, never blended within a single shape. Step 3 in Orange with dark text on the numbered circle (since white-on-orange fails contrast for small numerals). Do not reorder — the sequence reads as escalation/forward-motion the same way the Wing does.

**Connector arrows:** Raspberry `→` between Step 1→2, Raspberry `→` between Step 2→3 (the arrows stay Raspberry regardless of card color so they read as the same flow primitive).

**Speaker notes (optional):** "The third step is where the magic happens — but the second step is where the model earns its keep. If your description is wrong, scanning matches the wrong skill, and you load garbage into context."

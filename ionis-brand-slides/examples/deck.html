SKILLZ
01 · The Gap

Prompts forget. Agents stay home.

We've got great tools for one-shot tasks and great tools for in-platform agents. But there's a missing layer in between — the part that travels with you.

Prompt

Powerful, but one-shot.

A great prompt solves one problem in one chat. Close the window and it's gone. Share it as a screenshot and it rots.

Agent

Useful, but stuck.

Agents live inside one platform (Glean, ChatGPT, Claude…). When the tooling changes, the agent doesn't come with you.

SKILLZ
02 · Definition

A skill is a reusable instruction package the AI can load when it's relevant.

skillz /skɪlz/ · noun · same as "skills" — we just spell it with a z.

Think of it as a folder the model can open on demand — with a how-to guide inside, plus templates, examples, and rules. Portable across LLMs. Owned by you, not the platform.

"The skillz you create are the only portable artifacts that will serve you across LLM solutions." — practical framing we've been working with.
SKILLZ
03 · The Idea

Conditional memory — loaded only when it's needed.

Your AI doesn't read every skill all the time. It scans the names and descriptions of skillz available to it, then loads the relevant one(s) into context for that task — and multiple skillz can compose for a single request.

04 · Anatomy

What's inside a skill.

---
name: ionis-brand-slides
description: Use when creating slide decks,
  PowerPoints, or visual presentations that
  should follow Ionis brand standards —
  colors, typography, the Prow.
---

# Instructions
1. Use Raspberry (#A20067) as the key color.
2. Pair with Magenta (#D70073) or Orange.
3. Keep secondary tints for accents only.
4. Default typeface: Gotham, fallback Arial.
5. Always reserve white space — it's the foil.

# Resources
- /brand/colors.json
- /brand/logo-lockups/
- /templates/title-slide.pptx

Name

A short, descriptive identifier. The model uses this as part of its decision to invoke.

Description

The "when to use me" line. This is the most important part — the model reads it to decide if your skill is relevant to the task.

Instructions

The actual know-how. Rules, steps, dos and don'ts. Written in plain English.

Resources (optional)

Templates, examples, JSON, images — anything the skill can pull in when it runs.

05 · The Trigger

How the AI decides when to call a skill.

It's a matching problem, not a magic problem. The model sees the user's request, then scans skillz descriptions for a fit.

1

You ask.

"Build me a deck for the Q3 review." The model now has a task.

2

It scans.

The model reads each skill's description — never the full body. Cheap, fast.

3

It loads.

If a description matches the task, the skill's instructions and resources are pulled into context for that run.

The description is the contract. A good one says exactly when to use the skill — and just as importantly, when not to.

09 · How To

How to make & use one — today, with Glean.

You don't write skillz by hand. You ask Glean Assistant to write them for you. Two tabs in the browser: Platform (Glean-built, more coming) and Personal (yours). Focus today is Personal.

1

Go to Settings → Skills.

You'll see two tabs — Platform (Glean-published, more coming) and Personal. Stay on Personal.

→
2

Click "Create with Assistant."

Opens a Glean chat scoped to skill authoring. Glean asks clarifying questions, then scaffolds your SKILL.md in canvas.

→
3

Chat to refine. Save.

Iterate until the description and instructions are right. Save — it's live in your skillz library, routed by description or call by name.

To update an existing skill: download it from the browser, edit, then in the skill browser pick Replace on the skill you're updating and upload the new file. (Direct in-chat updates are coming soon.)

10 · Field Tips

Four things that actually matter.

01

Turn on Thinking Mode.

Glean's routing and execution are strongest in Thinking Mode. Use it when quality matters.

02

Same permissions as you.

Skillz inherit your Glean permissions. They never grant new data access — safe for compliance.

03

Call it by name.

Routing is automatic by description. For predictable behavior, say "use my X skill to…" — or type /skill-name as a slash command to invoke it explicitly.

04

Start with what you already repeat.

Meeting prep, weekly status, doc summaries, narrative drafts. If you do it weekly, it's a candidate.

12 · Why It Matters

Skillz are the durable layer.

Tools come and go. The skill stays with you, the team, and eventually — the company.

Portable

Plain text + files. Runs anywhere an LLM can read instructions — Glean today, something else tomorrow.

Composable

Stack them. "Draft Q3 email in my voice" loads my-style + q3-talking-points together — one task, many skillz.

Institutional

Captures the "how we actually do it here" knowledge that normally lives in one person's head.

Compounding

Skillz improve with feedback. Each fix, each new example, makes every future run a little better.

What know-how would you turn into a skill?

If you can describe the steps to a teammate, you can write a skill. Find one workflow you do over and over — that's your first one.
